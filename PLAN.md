# Cherry: VLM Spatial Understanding Evaluation Plan

This document outlines the research plan for evaluating Vision-Language Models' 3D spatial understanding through embodied interaction in Unity.

## Research Goal

**Test whether VLMs can accurately infer 3D spatial relationships from 2D images by having them externalize their understanding in a 3D environment.**

The key insight: Rather than asking VLMs to answer spatial questions textually (which may just test language priors), we have them "act out" their understanding by placing objects in a 3D scene—like a human reconstructing a room layout from memory.

## Core Concept: Scene Program

The VLM's output is a **scene program**—a sequence of tool calls that reconstruct the spatial layout:

```python
# Example scene program generated by VLM
place_object("sofa", x=-3, y=0.5, z=5)      # Large sofa on the left, background
place_object("coffee_table", x=0, y=0.5, z=3)  # Table in center, foreground
place_object("chair", x=4, y=0.5, z=4)      # Chair on the right
place_object("lamp", x=-5, y=1.5, z=6)      # Floor lamp, left side, elevated
place_object("painting", x=0, y=2.5, z=8)   # Painting on wall, center, far back
```

This program encodes the VLM's understanding of:
- **Left/Right relationships** (X axis)
- **Near/Far relationships** (Z axis / depth)
- **Above/Below relationships** (Y axis)
- **Relative sizes** (scale parameter)

## Evaluation Metrics

| Metric | Description | How Computed |
|--------|-------------|--------------|
| **Coverage** | Fraction of ground-truth entities placed | `placed / total_entities` |
| **Left/Right Accuracy** | Pairwise horizontal relations preserved | For each pair (A,B): does `A.x < B.x` match ground truth? |
| **Near/Far Accuracy** | Pairwise depth relations preserved | For each pair (A,B): does `A.z < B.z` match z_order? |
| **Above/Below Accuracy** | Pairwise vertical relations preserved | For each pair (A,B): does `A.y < B.y` match expected? |

## Implementation Phases

### Phase 1: Model Provider Abstraction

**Goal**: Create unified interface for multiple VLMs to unblock `run_unity_eval.py`

**Files to create**:
```
spatial_agent/
├── __init__.py
├── model_providers.py    # VLMProvider base class + implementations
└── prompts.py            # System prompts and query templates
```

**Supported providers**:
- `ClaudeProvider` - Anthropic Claude (claude-sonnet-4-20250514, opus)
- `OpenAIProvider` - GPT-4o, GPT-4-turbo
- `QwenProvider` - Qwen-VL via vLLM or Ollama

**Interface**:
```python
class VLMProvider:
    def generate(
        self,
        image_path: str,
        prompt: str,
        system_prompt: str,
        tools: List[Dict],
    ) -> VLMResponse:
        """Generate response with optional tool calls."""
        ...

@dataclass
class VLMResponse:
    text: str
    tool_calls: List[ToolCall]

@dataclass
class ToolCall:
    name: str
    arguments: Dict
```

### Phase 2: Tighten Evaluation Loop

**Goal**: Ensure spatial graph and Unity evaluation are properly connected

**Tasks**:
1. Fix schema mismatch: spatial graph `nodes` → evaluation `entities`
2. Verify coordinate mapping is correct (image pixels → Unity coordinates)
3. Add detailed per-relation logging for debugging
4. Consider adding embodied verification (VLM can use `capture_view` to verify placements)

**Coordinate System Alignment**:
```
Spatial Graph (2D + depth)          Unity (3D)
──────────────────────────          ──────────
bbox_center[0]: 0..width    ──►     X: -10..+10  (left/right)
bbox_center[1]: 0..height   ──►     Y: inferred  (height based on category)
relative_depth: 0..1        ──►     Z: -10..+10  (near/far)
```

### Phase 3: Experiment Runner

**Goal**: Run systematic experiments across models and images

**Features**:
- Batch processing of multiple images
- Multiple model comparison (Claude vs GPT-4o vs Qwen)
- Aggregated metrics and reporting
- Optional: vary prompts/queries to test different aspects

**Usage**:
```bash
# Run full experiment
python run_experiment.py \
    --images photos/*.jpg \
    --providers claude openai qwen \
    --output-dir experiments/exp_001

# Generates:
#   experiments/exp_001/
#   ├── results.json          # Aggregated metrics
#   ├── per_image/            # Per-image results
#   └── analysis.html         # Visual report
```
